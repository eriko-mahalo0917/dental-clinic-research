## 1. プロジェクト全体の構成と流れ

### ファイル構成

```
dental-clinic-research/
├── main.py                    # 起動ファイル（入口）
├── main_flow.py               # 全体の流れを管理する

├── .env                       # APIキーなど秘密情報（Gitに上げない！）
├── creds.json                 # Google APIの認証ファイル（Gitに上げない！）
├── requirements.txt           # 使用ライブラリ一覧
│
├── google_apis/
│   ├── sheets_reader.py       # スプシからデータを読み込む
│   ├── sheets_writer.py       # スプシにデータを書き込む
│   └── gmaps_api.py           # Google Maps APIでデータを取得
│
├── models/
│   └── clinic_data_flow.py    # 取得データを1行分の辞書にまとめる
│
└── utils/
    ├── logger.py              # ログ表示・保存の設定
    ├── popup.py               # 完了通知のポップアップ
    └── path_helper.py         # ファイルパスを返す（exe対応）
```

### 処理の全体フロー（5ステップ）

```
【1】スプシからクリニック名を取得
      ↓ gspread でシートを読む → pandas の DataFrame に変換
      ↓ 重複を削除（drop_duplicates）
      ↓ ステータスが空白の行だけ抽出

【2】Google Maps API でクリニック情報を取得（1件ずつ）
      ↓ Text Search でクリニック名を検索 → place_id を取得
      ↓ Place Details API で住所・電話番号・評価・口コミを取得

【3】取得データを1行分の辞書にまとめる
      ↓ ClinicDataFlow.make_sheet_data() でスプシに書ける形にする

【4】Google Sheets API でスプシにデータを書き込む
      ↓ 新しいワークシートを作成（batchUpdate）
      ↓ セルにデータを書き込む（batchUpdate）
      ↓ クリニック一覧のステータスを「WS作成済み」に更新

【5】完了ポップアップを表示
      ↓ PySide6 で「XX件完了しました」と通知
```

### ④ Google Sheets API（batchUpdate）

```python
from googleapiclient.discovery import build

# APIクライアントを作成（"sheets" = Sheets API, "v4" = バージョン）
service = build("sheets", "v4", credentials=creds)

# batchUpdate の基本構造
body = {
    "requests": [
        # ここにリクエストの辞書を複数入れると1回のAPIで全部実行される
        {"addSheet": {"properties": {"title": "新しいシート名"}}},
        {"updateCells": { ... }},
    ]
}

# 実行（execute() を忘れずに！）
response = service.spreadsheets().batchUpdate(
    spreadsheetId="スプシのID",
    body=body
).execute()
```

> **重要：batchUpdate とは**
> - 複数の操作（シート作成、セル書き込みなど）を **1回のAPIで** まとめて実行できる仕組み
> - 1操作ずつAPIを呼ぶと遅くなるのを防ぎ、API制限にも引っかかりにくい

### ⑤ Google Maps API（requests ライブラリ）

```python
import requests

# Text Search：クリニック名で検索して place_id を取得
url = "https://maps.googleapis.com/maps/api/place/textsearch/json"
params = {"query": "���〇歯科", "language": "ja", "key": api_key}
response = requests.get(url, params=params, timeout=10)

# エラー判定（HTTPエラーなら例外を投げる）
response.raise_for_status()

# JSONを辞書に変換
data = response.json()

# 辞書から安全に値を取り出す（キーがなければ第2引数を返す）
results = data.get("results", [])  # なければ空リスト
place_id = results[0].get("place_id")  # なければ None
```

> **重要：`.get()` の使い方**
> - `辞書["キー"]` はキーがないと **エラー（KeyError）** になる
> - `辞書.get("キー", デフォルト値)` はキーがなくてもエラーにならず、デフォルト値が返る
> - APIのレスポンスは予期しない構造が返ってくることがあるので、必ず `.get()` を使う

## 4. ハマった・注意すべきポイント

### ① Mac専用ライブラリは Windows でインストールエラーになる

**問題**
- `pyobjc-framework-Cocoa` や `pyobjc-core` は macOS 専用
- Windows で `pip install -r requirements.txt` すると失敗する

**解決策：環境マーカーを使う**

```
# requirements.txt に ; sys_platform == "darwin" を追加
pyobjc-framework-Cocoa==11.1; sys_platform == "darwin"
pyobjc-core==11.1; sys_platform == "darwin"
macholib==1.16.4; sys_platform == "darwin"
```

- `sys_platform == "darwin"` → macOS のときだけインストール
- Windows では自動的にスキップされる

---

### ② Mac専用コードも Windows でエラーにしないようにする

**問題**
- `from AppKit import NSApp` は Mac 専用
- Windows で実行するとエラーになる

**解決策：try/except + sys.platform で二重保護**

```python
import sys

try:
    if sys.platform == 'darwin':
        from AppKit import NSApp, NSApplicationActivateIgnoringOtherApps
except ImportError:
    pass  # ライブラリがなくても止まらないようにする

# 使う箇所でも同様にチェック
if sys.platform == 'darwin':
    try:
        NSApp.activateIgnoringOtherApps_(True)
    except NameError:
        pass
```

> **sys.platform の値**
> - `'darwin'` → macOS
> - `'win32'` → Windows（64bitでも）
> - `'linux'` → Linux

### ⑦ exe ファイル（PyInstaller）での実行時のパス問題

```python
from pathlib import Path
import sys

# PyInstaller で固めたexeファイルからは __file__ が使えない場合がある
# sys.frozen 属性で exe かどうか判定する
if getattr(sys, 'frozen', False):
    # exe として実行中 → exe のある場所を root にする
    root = Path(sys.executable).parent
else:
    # Python で直接実行中 → ファイルの2つ上を root にする
    root = Path(__file__).parents[1]
```

## 5. よく使うパターン集

### enumerate() で「���号」と「値」を同時に取り出す

```python
# 普通の for はインデックスが取れない
for name in name_list:
    print(name)  # 値だけ

# enumerate を使うと番号も取れる
for index, name in enumerate(name_list):

# 行インデックス +1 にして、シートの正しい行に書き込む例
for row_index, row_data in enumerate(clinic_list_rows):
    # row_index=0 → 1行目（ヘッダー）なので +1 してデータ行を指す
    "rowIndex": row_index + 1
```

### Optional を使った None チェック

```python
from typing import Optional

def get_place_id(self, result: Optional[Dict]) -> Optional[str]:
    # まず None チェックを最初にしておく（これをしないと後でエラーになる）
    if result is None:
        return None

    results = result.get("results", [])
    if not results:      # 空リストも False として扱われる
        return None

    place_id = results[0].get("place_id")
    if not place_id:    # None も False として扱われる
        return None

    return place_id
```
